Spring學習筆記03-前後端請求與回應

By 巫宗霖 2021-02-21

Spring 前後端請求與回應 簡單筆記。

Http 協議：規定前後端程式之間的資料傳輸格式。

格式如下：

Http Request (請求) Http Response(回應)
http method (get,post,put…等) http status code (200,404 … 等)
url (網址) response header
request header response body (後端回傳資料會在這裡)
request body (前端json資料通常放這裡)

HTTP 狀態碼 (http status code)

http 狀態碼依照首位數字不同分為五大類。

狀態碼 常見範例
1xx：資訊 很少用
2xx：成功 200 (請求成功)，201(請求成功且資源被創建)，202(請求成功，但後端尚未處理完成)
3xx：重新導向 301 (永久重新導向，通常出現在網站搬家)，302(暫時重新導向)
4xx：前端請求錯誤 400 (前端參數請求錯誤)，401 (沒通過身分驗證)，403 (請求遭到拒絕)，404 (網站不存在)
5xx：後端處理錯誤 500 (後端發生錯誤，可能為bug導致)

URL 路徑對應

使用標籤@RequestMapping，範例如下：

當前端發送請求至 url：/student 時，可以用以下方法對應。

@RequestMapping("/student")
public String testURL(){
	return "success";
}

配合Rest風格可以限制前端必須使用何種 http method 發送請求。

範例限制必須以GET來發送請求：

@RequestMapping(value = "/students", method = RequestMethod.GET)
public String testURL(){
	return "success";
}

若覺得上面這樣設定太麻煩也能直接使用@GetMapping，@PostMapping，@PutMapping，@DeleteMapping標籤代替@RequestMapping。

修改上方範例：

@GetMapping("/student")
public String testURL(){
	return "success";
}

即可得到相同效果。

取得 Request 參數方法

  1. 當參數放在url路徑內的問號後面時，url：/students?id=11040303

    使用 @RequestParam 取得參數，且參數名稱需與url中的名稱相同。也可以使用name=“參數名"來指定要的資料，但不建議。

    範例：

    url：/product?id=41252

    @RequestMapping("/product")
public String testRequestParam(@RequestParam Integer id){
	System.out.println("商品編號為："+id);
	return "success";
}

  2. 當參數放在request body 時，使用 @RequestBody 取得 requestbody 內的 json 內容。

    範例：

    url：/testBody

    requestbody：{“id”:123, “name”: “Amy” }

    class Student{
	private Integer id;
	private String name;
	...
	getter and setter
	...
}

    @RequestMapping("/testBody")
public String testRequestBody(@RequestBody Student student){
    System.out.println("id: " + student.getId());
    System.out.println("name: " + student.getName());
    return "success";
}

  3. 當參數放在request header 時，使用 @RequestHeader 取得 requestHeader 內的參數。

    範例：

    url：/testHeader

    requestheader：Content-Type:application/json

    @RequestMapping("/testHeader")
public String testHeader(@RequestHeader(name="Content-Type") String info){
    System.out.println("info: " + info);
    return "success";
}

    由於Java內需使用駝峰式命名法，但Content-Type不屬於此類，因此使用name=“Content-Type” 來直接指定要這個header資料。

  4. 當參數直接放在url路徑上時，使用@PathVariable來取得路徑上的參數。

    範例：

    url：/testUrl/4104321/Judy

    @RequestMapping("/testUrl/{id}/{name}")
public String testPathVariable(@PathVariable Integer id,@PathVariable String name){
    System.out.println("Path id: " + id + ", Path name: " + name);
    return "success";
}
