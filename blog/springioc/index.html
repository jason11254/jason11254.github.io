Spring學習筆記01-IoC簡介

By 巫宗霖 2022-02-10

IoC (Inversion of Control) - 控制反轉，將object的控制權交給Spring容器管理。

由Spring容器管理的object則稱為bean。

Spring IoC簡易流程

  1. 在目標class上加上@Component：

    @Component只能加在class上，使用後即可將該class變為Spring容器管理的bean。

@Component
public class Apple {
    public void print(String message){
        System.out.println("這顆蘋果，"+message);
    }
}
  1. 在需要該物件的class上也加上@Component：
@Component
public class Person {
    ...
}

  1. 最後使用@Autowired將bean注入物件，完成DI (Dependency Inject) - 依賴注入。

    @Autowired只能加在物件上，用於取得Spring容器中的bean。

    範例中取得了Spring容器中Apple的bean，注入物件apple。

@Component
public class Person {
    @Autowired
    private Apple apple;
    
    public void myApple(){
        apple.print("是我的。");
    }
}

  1. 創建main運行myApple()方法後即可看到console上的運行結果：這顆蘋果，是我的。

    而使用Spring容器建立的bean注入至物件中即是IoC的流程。

Spring IoC 優點：

  1. 使程式之間關聯性降低，在修改上可以不用到處尋找其他有關聯到的地方。
  2. object的生命週期由Spring容器管理，無須工程師手動操作。
